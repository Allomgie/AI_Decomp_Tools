# Technical Environment Format Specification
# Semantic Expert IR – v10

Reference documentation for the JSON format used as technical context
for MIPS-to-C decompilation. Generated by `techenv_builder.py`.

---

## Design Philosophy

- **No inference without evidence** – every field reflects something
  directly observable in the ASM or header files. No guessing.
- **Sparse by default** – fields that add no signal are omitted
  (`flags` absent = all false, `globals` absent = no global access).
- **Self-contained per function** – a function entry plus its `env`
  block contains everything needed to reconstruct the C code.
- **Token-efficient** – the format is optimised for LLM context windows.
  Redundancy is eliminated; common patterns are compressed.

---

## Sources

| JSON field | Source |
|------------|--------|
| `env.s`    | `.h` header files (struct definitions) |
| `env.sym`  | `.h` header files (global variable declarations) |
| `env.ext`  | `.h` header files (extern function declarations) |
| `env.e`    | `.h` header files (enum definitions) |
| `funcs[].n`, `sf`, `stk`, `mem`, `br`, `calls`, `globals`, `flags` | `.s` ASM file |
| `funcs[].args`, `ret` | `.h` if declared in `env.ext`, otherwise inferred from `.s` |

---

## Top-Level Structure

```json
{
  "file":  "filename.s",
  "env":   { "s": {}, "sym": {}, "ext": {}, "e": {} },
  "funcs": []
}
```

## `env` – Static Type Environment

### `env.s` – Struct layouts

```json
"S0": {
  "0x0": "s32",
  "0x4": "u32 *",
  "0x8": "struct S1"
}
```

- Key: bare struct name (`ALCSeq`, not `struct_ALCSeq`)
- Value key: hex byte offset of the field
- Value: C type string of that field
- Offsets follow MIPS32 natural alignment (max 4-byte boundary)
- `"struct S1"` at an offset means S1 is **embedded by value** (not a pointer)

### `env.sym` – Global variable types

```json
"g_28":  "u32",
"g_604": "u16 *"
```

- File-scope variables only (no stack locals, no function parameters)
- Pointer types include `*` suffix

### `env.ext` – External function signatures

```json
"alCSeqNew":  "(void) -> s32",
"memcpy":     "(void *, void *, s32) -> void *",
"printf":     "(s8 *, ...) -> s32"
```

- Format: `"(param_types) -> return_type"`
- `void` in param position = no parameters
- `...` = variadic

### `env.e` – Enum definitions *(new in v10)*

```json
"AudioError": {
  "0": "AUDIO_OK",
  "1": "AUDIO_ERR_INIT",
  "-1": "AUDIO_ERR_NULL"
}
```

- Key: enum type name as declared in the header
- Value: mapping from integer value to enumerator name
- Only enums whose type name is actually referenced in the header/source
  are included (avoids pollution from unused common headers)
- Negative values are preserved as string keys

## `funcs[]` – Per-Function Analysis

### Core fields

| Field | Type   | Meaning |
|-------|--------|---------|
| `n`   | string | Function name (from ASM `glabel`) |
| `sf`  | int    | Stack frame size in bytes |
| `ret` | string | Return type: exact type string if declared in `env.ext`; `"v0"` if $v0 is written but type unknown; `"void"` if nothing is returned |

### `args` – Input parameters

**When declared in `env.ext` (ground truth):**
```json
"args": [
  {"reg": "a0", "type": "ALCSeq *"},
  {"reg": "a1", "type": "s32"}
]
```

**When not declared (heuristic from ASM):**
```json
"args": ["a0", "a1"]
```
Registers $a0–$a3 that were read before being written at function
entry. Always contiguous from a0 (gaps below the highest detected
argument are filled automatically to avoid under-counting).

---

### `stk` – Stack layout

```json
"stk": {
  "saved": {
    "ra": "0x2c",
    "s0": "0x18",
    "s2": "0x20"
  },
  "locals": {
    "blocks":  [{"off": "0x10", "size": 48}],
    "singles": 3
  }
}
```

**`saved`**: callee-saved registers → hex stack offset.
Registers: `ra`, `s0`–`s8`, `gp`.

**`locals.blocks`**: contiguous local variable regions (slots with
≤8-byte gaps merged). Each block:
- `off`: lowest offset (hex)
- `size`: total byte span

Large blocks (>16 bytes) typically indicate local arrays, structs,
or compiler-generated temporary buffers.

**`locals.singles`**: count of isolated scalar slots that did not
merge into any block.

### `mem` – Memory access patterns

One entry per unique base pointer used in the function:

```json
{
  "base":     "s2",
  "off":      "0x295c",
  "fields":   [
    {"off": "0x0",  "rw": "r"},
    {"off": "0x4",  "rw": "w"},
    {"off": "0x8",  "rw": "rw"}
  ],
  "size":     12,
  "struct":   "ALCSeq",
  "packed":   true,
  "in_block": true
}
```

| Field     | Present when          | Meaning |
|-----------|-----------------------|---------|
| `base`    | always                | Pointer source: register name, symbol name, or `"sp"` |
| `off`     | `base` is `"sp"`      | Stack offset of the pointer (hex) |
| `fields`  | always                | Accessed field offsets with access mode |
| `size`    | ≥2 fields             | Byte span: `max_offset − min_offset + 4` |
| `struct`  | pattern match found   | Struct name from `env.s` matching the access pattern |
| `packed`  | non-4-byte-aligned field | Non-naturally-aligned access (u8/u16 or packed struct) |
| `in_block`| `base` is `"sp"`      | This stack slot falls within a `stk.locals` block |

#### `fields[].rw` – Access mode (always present)

| Value  | Meaning |
|--------|---------|
| `"r"`  | Read-only. Field was only loaded from (`lw`, `lh`, `lb`, etc.) |
| `"w"`  | Write-only. Field was only stored to (`sw`, `sh`, `sb`, etc.). Typical for initialisation or output parameters. |
| `"rw"` | Read-write. Field was both loaded and stored. Typical for state mutation. |

`rw` is **always present** on every field entry. There are no implicit defaults.

---

### `globals` – Global variable access summary

```json
"globals": {
  "rand_state": "rw",
  "g_28":       "r",
  "g_47":       "w"
}
```

Tracks which `env.sym` globals this function reads or writes.
Same `"r"` / `"w"` / `"rw"` semantics as `fields[].rw`.
Key absent = function does not access any known globals.

---

### `calls` – Function calls made

```json
[
  {
    "type":  "direct",
    "name":  "alCSeqNextEvent",
    "argc":  1,
    "ctx":   {"a0": "s2+0x0"}
  },
  {
    "type":  "direct",
    "name":  "exit",
    "argc":  1,
    "tail":  true
  },
  {
    "type":  "indirect",
    "reg":   "t9",
    "hint":  "symbol",
    "argc":  2
  }
]
```

| Field  | Meaning |
|--------|---------|
| `type` | `"direct"` = `jal target` or tail-call `j target`; `"indirect"` = `jalr $reg` |
| `name` | Target function name (direct only) |
| `reg`  | Register holding the function pointer (indirect only) |
| `hint` | How the register was loaded: `"symbol"` = via `lui`/`addiu` from a named symbol; `"fp"` = loaded from memory (function pointer table); `"reg"` = untracked |
| `argc` | Number of $a0–$a3 registers set before this call |
| `ctx`  | Contents of argument registers: `"symbol+offset"` or symbol name. Only present when derivable from register tracking. |
| `tail` | *(new in v10)* `true` if this call is a tail call (`j target` instead of `jal`). Only present for direct calls. |

---

### `br` – Branch instruction summary

```json
[
  {"op": "beq",  "loop": true},
  {"op": "beql", "likely": true, "loop": true},
  {"op": "bne"}
]
```

One entry per unique branch opcode in the function.

| Field    | Present when                  | Meaning |
|----------|-------------------------------|---------|
| `op`     | always                        | MIPS branch opcode |
| `likely` | opcode ends in `l` (e.g. `beql`, `bnel`) | MIPS branch-likely: delay slot only executes if branch taken |
| `loop`   | at least one instance is a backward branch | This opcode was used as a loop back-edge at least once |

Absent `likely` = false. Absent `loop` = false.

---

### `flags` – Instruction class indicators

```json
"flags": {
  "has_fpu":   true,
  "has_div":   true,
  "has_mul":   true,
  "has_64bit": true
}
```

Only `true` flags are emitted. Key absent entirely = all false.

| Flag        | Meaning |
|-------------|---------|
| `has_fpu`   | Uses FPU instructions (float/double arithmetic or conversion) |
| `has_div`   | Uses integer division (`div` / `divu`) |
| `has_mul`   | Uses integer multiplication (`mult` / `mul`) |
| `has_64bit` | Uses 64-bit operations (`dadd`, `dmult`, `ld`/`sd`, `ldc1`/`sdc1`, etc.) |

## Type Strings

Primitive types used throughout:

| String | C equivalent |
|--------|-------------|
| `s8`   | `signed char` |
| `u8`   | `unsigned char` |
| `s16`  | `signed short` |
| `u16`  | `unsigned short` |
| `s32`  | `signed int` |
| `u32`  | `unsigned int` |
| `s64`  | `signed long long` |
| `u64`  | `unsigned long long` |
| `f32`  | `float` |
| `f64`  | `double` |

Pointer: append `*` (e.g. `s32 *`).
Array: append `[N]` (e.g. `u8 [16]`).
Const: prepend `const` (e.g. `const u32`).
